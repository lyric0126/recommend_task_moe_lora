What was wrong
- First MoCLE run trained on a regenerated interaction prompt format but was evaluated on HydraLoRA valid prompts. That was not the same standard and produced 26.17% accuracy.
- Second issue: MoCLE was using fixed lr=1e-5, no warmup/cosine scheduler, grad_accum=1. HydraLoRA used lr=2e-4, cosine, warmup_ratio=0.03, grad_accum=8.
- Third issue: MoCLE DataLoader used shuffle=False. HydraLoRA Trainer shuffles train data; with cosine LR, fixed ordering biases which domains see high vs low LR.

Final same-standard setup
- Train data: HydraLoRA data/hydralora_pseudo_v2fix_all/train.json, copied unchanged except cluster_id added from domain.
- Valid data: same HydraLoRA valid split, copied unchanged except cluster_id added from domain.
- Routing: movielens->expert_0, goodreads->expert_1, mind->expert_2, kuairec->expert_3.
- Optimizer: AdamW, lr=2e-4, cosine, warmup_ratio=0.03, grad_accum=8, shuffle=True, one epoch-equivalent.
